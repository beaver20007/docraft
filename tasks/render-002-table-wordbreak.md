# render-002 — table_renderer рвёт слова в узких колонках

**Submodule:** `apps/ai-playbook` → `beaver20007/ai-playbook-generator`, `packages/render-engine`
**Класс:** YELLOW (правка рендера + PR). Мерж — только Оркестратор.
**Статус:** QUEUED → В РАБОТЕ. Продолжение render-001 (тот чинил KPI-карточки `stats_grid`).

## Откуда задача (факт от владельца, координаты из прод-PDF job caee3bb7)

`table_renderer` рвёт слова посимвольно в **узких колонках**. Утверждение render-001 «таблица
слова не рвёт» **опровергнуто артефактом** — воркер тестировал не на узкой 4-колоночной таблице.

**4-колоночная таблица KPI (стр.5 прод-PDF) — разрывы:**
- «Sales M»/«anager» x=78
- «15 квалифицирован»/«ных встреч» x=151
- «30% конв»/«ерсия в» x=294

**Контроль: 3-колоночная таблица (стр.2) те же строки НЕ рвёт.** → дело в **доступной ширине
колонки**, не в шрифте. При 4 колонках на ту же ширину страницы колонка уже → 12pt-текст
(`typography.scale.small` темы premium) не влезает → посимвольный перенос.

## Причина (разобрана Оркестратором — контекст, не гадать)

Кегль ячейки = `typography.scale.small` = **12pt** в premium (реальное значение темы; дефолт 8).
Не регрессия конкретного коммита: `pdf-001` починил рендер premium (был пустой PDF/0 стр.),
проявив 12pt-ячейки. Тот же класс бага, что render-001 закрыл для карточек. Чинить раскладку,
не кегль темы.

## Цель (DoD)

1. Воспроизвести на 4-колоночной таблице с «Sales Manager»/«30% конверсия» (координаты выше),
   извлечь факт из PDF (PyMuPDF: разрыв, ширина колонки, шрифт/кегль).
2. Починить по образцу render-001, но для таблиц: **запрет посимвольного переноса**
   (`splitLongWords=False` / убрать `wordWrap="CJK"` если есть) + перенос по словам и/или
   ужатие кегля по реальным метрикам `stringWidth` под ширину колонки. Ширины колонок —
   осмысленно (не фикс. поровну, если длинные значения не влезают).
3. Контроль: 3-колоночная таблица (которая НЕ рвала) не изменилась/не ухудшилась.
4. Кириллица и все 7 тем не сломаны (тот же прогон, что render-001).
5. **Свип остального (пункт владельца «что ещё переполняется»):** проверить `module_banner`
   (читает `scale.small`/`scale.body` в возможно узком баннере) и прочие компоненты со
   `scale`-токенами в стеснённой ширине. Найденное — в отчёт; чинить в этом PR только таблицу,
   остальное отдельными задачами если есть.
